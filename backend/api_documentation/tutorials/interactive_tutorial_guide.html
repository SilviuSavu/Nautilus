Welcome to the Nautilus Trading Platform! This tutorial will guide you through your first steps, from setup to making your first API call.

Step 1: Environment Setup

First, let's make sure you have everything you need to start using the Nautilus API.

Prerequisites:
  • Docker and Docker Compose installed
  • Basic knowledge of REST APIs
  • A code editor of your choice
🐳 Docker Setup 
# Clone the repository
git clone https://github.com/SilviuSavu/Nautilus.git
cd Nautilus

# Start all services
docker-compose up -d

# Check if services are running
docker-compose ps
🔍 Test Your Setup
Click "Test Connection" to verify your Nautilus API is running

Step 2: Your First API Call

Let's make your first API call to check the system health.

🌐 Health Check API Call 
curl http://localhost:8001/health

You should receive a response like this:

📋 Expected Response 
{
  "status": "healthy",
  "timestamp": "2025-01-23T10:30:00.000Z",
  "version": "3.0.0",
  "components": {
    "database": {"status": "up", "response_time": 5.2},
    "redis": {"status": "up", "response_time": 1.1}
  }
}

Step 1: Understanding JWT Authentication

Nautilus API uses JWT (JSON Web Tokens) for secure authentication. Here's how it works:

Authentication Flow:
  1. Send login credentials to /api/v1/auth/login
  2. Receive access_token and refresh_token
  3. Include access_token in all API requests
  4. Use refresh_token to get new access_token when it expires
🔑 Login Request 
curl -X POST http://localhost:8001/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "trader@nautilus.com",
    "password": "your_secure_password",
    "remember_me": false
  }'
🔓 Try Authentication
Click "Login Demo" to see authentication in action

Step 2: Using Access Tokens

Once you have an access token, include it in the Authorization header for all authenticated requests.

🎫 Authenticated Request 
curl -X GET http://localhost:8001/api/v1/system/metrics \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN_HERE"
⚠️ Security Best Practices:
  • Never hardcode tokens in your application
  • Store tokens securely (encrypted storage)
  • Implement automatic token refresh
  • Use HTTPS in production environments

Step 3: Token Refresh Pattern

Access tokens expire for security. Implement automatic refresh using the refresh token:

🔄 Token Refresh 
curl -X POST http://localhost:8001/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "YOUR_REFRESH_TOKEN_HERE"
  }'

Step 1: Real-time Quotes

Get live market data from multiple sources with automatic fallback.

📊 Get Real-time Quote 
curl -X GET "http://localhost:8001/api/v1/market-data/quote/AAPL?source=IBKR" \
  -H "Authorization: Bearer YOUR_TOKEN"
