Step 1: Understanding JWT Authentication

Nautilus API uses JWT (JSON Web Tokens) for secure authentication. Here's how it works:

Authentication Flow:
  1. Send login credentials to /api/v1/auth/login
  2. Receive access_token and refresh_token
  3. Include access_token in all API requests
  4. Use refresh_token to get new access_token when it expires
üîë Login Request 
curl -X POST http://localhost:8001/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "trader@nautilus.com",
    "password": "your_secure_password",
    "remember_me": false
  }'
üîì Try Authentication
Click "Login Demo" to see authentication in action

Step 2: Using Access Tokens

Once you have an access token, include it in the Authorization header for all authenticated requests.

üé´ Authenticated Request 
curl -X GET http://localhost:8001/api/v1/system/metrics \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN_HERE"
‚ö†Ô∏è Security Best Practices:
  • Never hardcode tokens in your application
  • Store tokens securely (encrypted storage)
  • Implement automatic token refresh
  • Use HTTPS in production environments

Step 3: Token Refresh Pattern

Access tokens expire for security. Implement automatic refresh using the refresh token:

üîÑ Token Refresh 
curl -X POST http://localhost:8001/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "YOUR_REFRESH_TOKEN_HERE"
  }'
